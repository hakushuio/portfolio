# portfolio

HAKUSHU.IO portfolio website.

This repository contains the source files and generated static assets for the
portfolio site.

## Overview

- Source files are located in `src`.
- Generated deployable files are located in `dist`.
- Gulp compiles Pug, Sass, JavaScript, and images from `src` into `dist`.
- The default Gulp task also starts BrowserSync and watches source files.

## Directory Structure

```text
.
├── dist/          # Generated static files for deployment
├── gulpfile.js/   # Gulp task definitions and module imports
├── src/           # Source files
│   ├── img/       # Source images
│   ├── js/        # Source JavaScript
│   ├── pug/       # Pug templates
│   └── scss/      # Sass stylesheets
├── package.json
└── README.md
```

## Requirements

This project was verified with the following environment.

```text
Node.js v14.15.1
Gulp CLI 2.3.0
Gulp 4.0.2
```

The current dependency tree uses `gulp-sass` 4.x, which depends on
`node-sass`. `node-sass` is sensitive to the Node.js runtime version, so newer
Node.js versions may fail before Gulp tasks start.

## Setup

```bash
npm install
```

## Development

```bash
npm start
```

This runs the default Gulp task:

- compiles Pug templates into HTML
- compiles Sass into CSS and minified CSS
- minifies and bundles JavaScript
- optimizes changed images
- starts BrowserSync from `dist`
- watches files under `src`

## Gulp Tasks

```bash
npx gulp html
npx gulp css
npx gulp js
npx gulp img
```

Task outputs:

- `html`: `src/pug/*.pug` to `dist/*.html`
- `css`: `src/scss/*.scss` to `dist/css/*.css` and `dist/css/*.min.css`
- `js`: `src/js/**/*.js` to `dist/js/main.min.js`
- `img`: `src/img` to `dist/img`

## Deployment

Deployable outputs generated by Gulp are located in the `dist` directory.

The deployment workflow publishes `dist/` to GitHub Pages. The repository Pages
source must be configured to GitHub Actions in the GitHub repository settings.

The production custom domain is configured in the repository GitHub Pages
settings.

## Notes

- `dist` is committed and represents the generated deployment output.
- `npm test` is currently a placeholder and exits with an error.
- Use the verified Node.js version above when rebuilding assets unless the Sass
  toolchain is upgraded.
